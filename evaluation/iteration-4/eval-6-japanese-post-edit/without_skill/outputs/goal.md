リポジトリを確認しました。`package.json` に `test:auth` スクリプト（`jest tests/auth`）があり、`tests/auth/login.test.ts` が `src/auth/login.ts` の `login` 関数をテストしています。この構成なら、以下の `/goal` で認証まわりのテストが全部通るまで自走させられます。

```
/goal npm run test:auth が exit code 0 で終了するまで、src/auth 配下の実装を修正し続けてください。テストファイル（tests/auth 配下）は変更しないでください。
```

補足:
- `test:auth` は `jest tests/auth` を実行するスクリプトなので、認証関連テストだけに絞って回せます（全体の `npm test` だと無関係な失敗にも足を取られる可能性があります）。
- 「テストファイルを変更しない」を明示しているのは、テストの中身を書き換えて通す方向に流れるのを防ぐためです。
- この `/goal` を実行すると、条件を満たすまで毎ターンの承認なしに自走します。危険な操作（`git push --force` やファイル削除など）は行わせない前提です。途中の差分は都度 `git diff` などで確認することをおすすめします。
- 実行前に `npm install`（もしくは `npm ci`）で依存関係が入っているか確認してください。

このまま実行して問題なければ、上記の `/goal` 行をそのままターミナルに貼り付けてください。
